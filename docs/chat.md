# AI Nutritionist Chat

## Overview

The Chat feature provides a conversational AI nutritionist powered by **Gemini 2.1 Flash Lite Preview**. It is context-aware, streaming-first, and automatically pulls in the user's real dietary history and profile to give personalised advice.

---

## Architecture

```
ChatScreen (UI)
    │
    ├── ChatService.buildDietContext()      ← Supabase: food_logs + profiles
    ├── ChatService.generateSuggestions()   ← Gemini: 4 personalised chip questions
    ├── ChatService.detectFoodQuery()       ← local keyword match
    ├── ChatService.searchCfsFood()         ← Supabase: CFS text search + vector fallback
    └── ChatService.streamChat()            ← Gemini SSE stream → Stream<String>
```

All LLM calls go through the OpenAI-compatible relay at `AppConfig.visionApiUrl` using `AppConfig.visionModel`.

## Features

### 1. 30-Day Diet Context

On screen open, `buildDietContext()` fetches:

- **User profile** from `profiles` table — name, gender, height, weight, activity level, goal type, daily macro targets, dietary restrictions, allergies
- **Food logs** from `food_logs` table — all entries in the past 30 days, ordered by `logged_at`

The data is formatted as a compact text block injected into the Gemini system prompt:

```
## User Profile
Name: Alice
Gender: Female
Height: 165cm
Weight: 58kg
Activity: moderately_active
Goal: lose_weight
Daily targets: 1600kcal, protein 120g, carbs 160g, fat 53g
Dietary restrictions: vegetarian

## Food Log (last 30 days, 42 entries)
Date | Meal | Food | Cal | P | C | F
2026-03-01 | breakfast | Oatmeal | 350kcal | 12g | 58g | 7g
...

Daily averages (28 days): 1542kcal, protein 108g, carbs 165g, fat 51g
```

### 2. Suggestion Chips

Before the user sends their first message, 4 personalised question chips are displayed. These are generated by Gemini based on the diet context:

- Prompt: *"Generate exactly 4 short suggested questions (≤15 words) the user might want to ask you. Return ONLY a JSON array of strings."*
- Temperature: 0.7, max_tokens: 500
- Fallback list displayed if the API call fails or times out
- Chips disappear after the user sends the first message

### 3. CFS RAG (Centre for Food Safety Database)

When a message contains nutrition-related keywords, `detectFoodQuery()` extracts the food name and triggers a database lookup against the Hong Kong CFS nutrition database.

**Trigger keywords** (Chinese + English):

> 卡路里 · 熱量 · 蛋白質 · 碳水 · 脂肪 · 營養 · 纖維 · 幾多卡 · calorie · protein · carb · fat · nutrition · kcal · macro · fiber · sugar · sodium

**Search strategy** (two-phase, zero cold-start):

1. **Text search** — `ilike food_name_chi '%<name>%'` against `cfs_foods`, returns up to 3 hits
2. **Vector fallback** — if text search returns nothing, generate an embedding via `AppConfig.embeddingApiUrl` and call the `match_cfs_food` Supabase RPC (cosine similarity threshold 0.45)

The matched records (energy, protein, carbohydrate, fat per 100g) are appended to the user's message before it is sent to Gemini, giving the model authoritative official data to cite.

When RAG is active a **"Searching food database..."** indicator appears in the status bar.

### 4. Streaming Response

`streamChat()` uses HTTP SSE (Server-Sent Events):

- Sends `"stream": true` in the OpenAI-compatible request body
- Reads the response via `http.Client().send()` → `StreamedResponse`
- Buffers partial lines, splitting on `\n`
- Parses `data: {"choices":[{"delta":{"content":"..."}}]}` lines and `yield`s each token
- Handles `data: [DONE]` to terminate the stream

In `ChatScreen._sendMessage()`, an empty assistant `_ChatMessage` is inserted, then each token is appended via `setState`, producing a live typing effect.

### 5. Thinking Indicator

While the assistant message has no content yet (SSE not yet delivered any tokens), the bubble shows an animated **"Thinking..."** widget (`_ThinkingDots`) — three dots fading in and out with staggered timing via `AnimationController`. A spinner + "Thinking..." label also appears in the status bar.

Once the first token arrives, the bubble switches to `MarkdownBody` for full Markdown rendering.

### 6. Markdown Rendering

Responses are rendered with `flutter_markdown` (`^0.7.7`). Supported formatting:

- **Bold** — nutritional highlights
- Bullet lists — food suggestions, meal plans
- Code blocks — macros tables if the model chooses to format them
- Blockquotes — styled with a primary-colour left border

## Data Flow (per message)

```
User types message
        │
        ▼
detectFoodQuery()  ──── match? ──→  searchCfsFood()  ──→  cfsContext (string)
        │                                                        │
        ▼                                                        │
Build message history (all prior turns)                         │
        │                                                        │
        ▼                                                        │
Append cfsContext to latest user message ◄───────────────────────
        │
        ▼
Insert empty assistant bubble (_ThinkingDots shown)
        │
        ▼
streamChat(messages, dietContext, cfsContext)
        │   SSE token stream
        ▼
setState → append token to assistantMsg.content
        │
        ▼  (first token received → bubble switches to MarkdownBody)
        ...
        ▼
onDone → _isStreaming = false
```

## System Prompt

```
You are an expert AI nutritionist integrated into the SmartDiet AI app.
You have access to the user's real dietary data shown below.
Reference specific dates and numbers when relevant.
Give concise, actionable advice.
Reply in the SAME LANGUAGE as the user's message.

<dietContext>
```

The model is instructed to **mirror the user's language** — if the user writes in Cantonese it replies in Cantonese, English in English, etc.

## Configuration

All API credentials and endpoints are in `AppConfig`:

| Key | Usage |
|-----|-------|
| `visionApiUrl` | Gemini relay base URL |
| `visionApiKey` | Bearer token |
| `visionModel` | `gemini-2.1-flash-lite-preview` |
| `embeddingApiUrl` | Embedding endpoint (vector fallback) |
| `embeddingApiKey` | Bearer token |
| `embeddingModel` | Embedding model name |
| `embeddingDimensions` | Vector dimensions |

## Supabase Tables Used

| Table | Columns read | Purpose |
|-------|-------------|---------|
| `profiles` | `display_name`, `gender`, `height_cm`, `weight_kg`, `activity_level`, `goal_type`, `target_calories`, `target_protein`, `target_carbs`, `target_fat`, `dietary_restrictions`, `allergies` | User context for system prompt |
| `food_logs` | `food_name`, `calories`, `protein`, `carbs`, `fat`, `meal_type`, `logged_at` | 30-day history |
| `cfs_foods` | `food_name_chi`, `food_name_eng`, `energy_kcal`, `protein_g`, `carbohydrate_g`, `fat_g` | CFS RAG source |

RPC: `match_cfs_food(query_embedding, match_count, match_threshold)` — pgvector cosine search.

## Error Handling

| Scenario | Behaviour |
|----------|-----------|
| API non-200 response | Stream throws; bubble shows "Sorry, something went wrong. Please try again." with error styling |
| Network timeout (90s) | Same as above |
| Profile missing | Skipped silently; context contains food logs only |
| No food logs | Context contains "No food logs in the past 30 days." |
| Suggestion API fails | Falls back to 4 hardcoded generic questions |
| CFS text + vector both empty | `cfsContext` is null; no RAG block appended |
